# 当前限制与发布范围

这份文档用于说明 XplorOne 当前发布阶段的使用范围。

它不是路线图，也不承诺未来功能。
它的作用是帮助用户理解：XplorOne 当前能做什么，哪些能力是可选的，哪些边界是有意保留的。

## 1. 当前正式发布线

XplorOne 当前正式发布线是桌面应用。

当前阶段：

- Windows 是当前正式发布线
- Electron 桌面端是面向终端用户的正式发布形态
- Web 预览仅用于开发和内部预览
- Web 预览不是正式终端用户发布形态

如果你是普通用户，请优先使用官方桌面版本，而不是开发预览路径。

## 2. 核心本地工作流不需要 API Key

即使没有模型 API Key，XplorOne 也可以继续使用。

你可以使用这些核心本地功能：

- 本地记账
- 记一笔
- 账本、账户、分类和流水
- 报表与财务视图
- 基础本地查询
- 当前版本支持的备份、恢复、导出、导入与归档工作流

模型 API Key 只在你需要 AI 辅助能力时才需要。

## 3. AI 功能是可选增强

AI 是增强层，不是 XplorOne 的使用前提。

即使没有 API Key，XplorOne 也可以作为本地优先财务工作台使用。

当你配置自己的 API Key 后，AI 可以帮助：

- AI 助手中的分析与解释
- 更开放的财务相关对话
- 明确进入 AI 工作流时的模型辅助理解
- 启用模型参与时的录入草稿辅助

在本地内核支持的范围内，本地助手的查询和录入工作流应当在没有模型 API Key 的情况下继续可用。

当前 AI 边界包括：

- AI 不会自动写入账本
- 涉及写入的操作必须由用户确认
- 不同模型服务商和模型的表现可能不同
- 某些自然语言解析能力会继续逐步完善
- 用户需要自行选择并信任自己的模型服务商

详细说明请见：[隐私与 AI 边界](./privacy-and-ai-boundaries_zh-CN.md) 与 [BYOK 配置说明](./byok-setup_zh-CN.md)。

## 4. 本地 API 与 MCP 是高级可选集成

本地 API 与 MCP 不是普通桌面端使用的前提。

它们主要服务于高级本地集成和外部 agent 工作流。

当前边界包括：

- 本地 API 与 MCP 访问属于 token 保护的本地访问路径
- MCP 当前为只读访问
- MCP 不暴露写入操作
- MCP 查询会经过 XplorOne 受控的本地 API 与查询链路
- MCP 不直接读取数据库
- MCP 围绕当前激活账本作用域设计
- 外部 agent 负责解释结构化查询结果

对于普通用户来说，本地记账、报表、记一笔和软件内 AI 工作流都不需要 MCP。

## 5. XplorOne 不是云端会计套件

XplorOne 以本地优先为产品方向。

它当前不是：

- 云端优先的会计套件
- 浏览器优先的 SaaS 产品
- 托管式多人会计平台
- 实时云端协作系统
- 面向所有团队、所有地区的完整企业财务系统

这是有意保留的产品方向。

XplorOne 更适合需要桌面财务工作台，并且在意本地数据与 AI 边界的用户。

XplorOne 当前不提供自动云同步。跨设备使用应通过备份、导出、归档工作流，或未来明确发布的同步能力来管理。

## 6. 当前还不是完整多平台发布

当前阶段，不应把 XplorOne 描述成完整跨平台产品。

当前公开口径是：

- Windows 桌面端是当前正式发布线
- macOS、Linux、移动端和浏览器正式用户版本，除非明确发布，否则不应默认为可用
- Web 预览不能替代桌面发布版本

如果平台支持对你的工作流很重要，请先查看当前 Release 页面。

## 7. 不是税务、法律或会计合规服务

XplorOne 帮助用户整理、查看和理解财务记录。

它不是：

- 法律建议
- 税务建议
- 持证会计建议
- 特定国家或地区的税务申报系统
- 专业会计师或税务顾问的替代品

用户仍需自行判断财务决策，并遵守所在地的合规要求。

## 8. 导入与导出覆盖范围会随版本变化

XplorOne 包含数据控制与可迁移相关工作流，但用户不应默认所有外部格式都已支持。

根据当前发布状态，可能支持的工作流包括：

- 备份
- 恢复
- 导出
- 导入
- 归档账本迁移
- `.xpl` 归档包工作流

当前边界包括：

- 不一定支持所有银行、支付平台或会计软件导出格式
- 导入后的数据应由用户自行检查
- 归档和导出工作流可能有范围限制
- 在不同机器之间迁移数据时，可能需要重新配置模型 API Key 或本地集成 token

隐私和凭据边界请见：[隐私与 AI 边界](./privacy-and-ai-boundaries_zh-CN.md)。

## 9. 凭据可迁移性有边界

XplorOne 会把模型 API Key 和本地集成 token 视为敏感的本地凭据。

这意味着：

- 模型 API Key 不应以明文形式写入账本数据库
- MCP client token 属于受保护的本地凭据
- 导出的账本文件不应暴露明文凭据
- full-app 备份工作流可能包含受保护的凭据元数据
- 受保护凭据备份可能依赖当前机器或操作系统凭据环境

如果你把 XplorOne 数据迁移到另一台机器，可能需要重新配置模型 API Key 或本地集成 token。

## 10. 双语 UI 与 AI 表现仍在持续完善

XplorOne 支持中文和英文界面，并且双语 AI 表现会继续改进。

当前可能存在的边界包括：

- 部分后端错误信息可能尚未完全本地化
- 专业图表本地化已经改进，但部分高级图表标签或技术提示仍可能需要继续打磨
- 模型生成内容会受到模型服务商影响
- 英文和中文自然语言理解在不同工作流中的完整度可能不完全一致
- 启用 AI 个性化后，个性化上下文应跟随当前回复语言，但模型服务商输出仍可能有差异

如果你遇到语言问题，建议反馈时提供：

- 当前界面语言
- 输入语言
- 如果涉及 AI，请提供模型服务商
- 你期望的输出语言
- 实际输出语言

## 11. 报表与图表有当前版本范围

XplorOne 包含财务报表和图表，但报表能力仍应放在当前发布范围内理解。

当前报表区域可能包括：

- 现金流
- 收支
- 资产负债
- 账户与分类视图
- 部分图表与趋势视图

高级图表的可见性或可用范围，可能会根据发布渠道、产品层级或当前产品导航而变化。

专业图表本地化已经改进，但报表和图表标签仍应放在当前版本范围内理解。

不应把 XplorOne 描述成完整企业级 BI 系统。

有些功能可能会出现在路线图、版本说明或社区讨论里。

除非某个功能已经存在于当前版本，并且明确作为可用能力发布，否则不应把它当成当前功能。

如需确认，请查看：

如果某个限制影响了你的工作流，你可以：

- 在 [Discussions](https://github.com/SimonZhangM/XplorOne/discussions) 提问
- 在 [Issues](https://github.com/SimonZhangM/XplorOne/issues) 提交明确 bug
- 查看 [Releases](https://github.com/SimonZhangM/XplorOne/releases) 确认当前版本能力
- 在可用时查看 [Roadmap](../ROADMAP.md) 理解方向性信息
