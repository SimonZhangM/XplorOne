# BYOK 配置说明

BYOK 是 **Bring Your Own Key** 的缩写。

在 XplorOne 中，它表示：你可以使用自己的模型 API Key，把 XplorOne 连接到你选择的大模型服务。

模型 API Key 是可选的。
即使没有 API Key，XplorOne 也可以用于核心本地记账、报表、财务视图和基础本地查询。

## 1. BYOK 用来做什么

模型 API Key 只在你需要 AI 辅助能力时才需要。

根据你启用的工作流，AI 辅助能力可能包括：

- AI 助手中的分析与解释
- 更开放的财务相关对话
- 明确进入 AI 工作流时的模型辅助理解
- 启用模型参与时的录入草稿辅助

本地助手查询仍是本地优先工作流，受支持的本地查询不需要 API Key。

你不需要模型 API Key，也可以使用：

- 本地记账
- 记一笔
- 账本、账户、分类和流水
- 报表与财务视图
- 基础本地查询
- 当前版本支持的备份、恢复、导出、导入与归档工作流

简单说：

> 即使没有 API Key，XplorOne 也可以作为本地优先财务工作台使用。
> API Key 只是用于在核心本地工作流之上启用 AI 辅助能力。

## 2. 配置前需要准备什么

配置模型服务前，请先准备：

- 服务商或 provider 名称
- base URL
- model name
- API Key

XplorOne 围绕 OpenAI-compatible 的模型服务配置方式设计，并可能提供 OpenAI、Google Gemini 等常见服务商的内置预设。

不同服务商可能会有不同的：

- endpoint 格式
- model name
- 速率限制
- 计费规则
- 返回行为
- 代理或网络环境要求

如果你不确定正确的 base URL 或 model name，请先查看对应模型服务商自己的文档。

## 3. 如何配置模型 API Key

打开 XplorOne，进入：

`设置 -> 模型服务`

然后完成配置：

1. 选择服务商预设，或选择自定义服务商
2. 填写 base URL
3. 填写 model name
4. 填写你的 API Key
5. 保存 AI 配置
6. 测试连接
7. 启用当前配置

XplorOne 会区分这几个动作：

- **保存 AI 配置**：保存当前草稿配置
- **测试连接**：检查当前配置能否调用模型服务
- **启用当前配置**：把当前配置设为 AI 工作流实际使用的模型

保存配置不一定等于已经启用。
如果 AI 功能没有使用你预期的模型，请检查当前启用的是哪一组配置。

## 4. 尝试第一次 AI 助手工作流

启用模型配置后，你可以在 Chat 中尝试 AI 助手工作流。

例如：

> 帮我总结这个月的收入和支出。

> 帮我解释为什么这个季度餐饮支出上升了。

> 制定下个月预算前，我应该先看哪些财务模式？

受支持的本地助手查询由本地查询内核处理，不需要模型 API Key。
当你明确进入模型辅助分析或更开放的财务相关对话时，才会使用 AI 助手。

## 5. API Key 保存方式

模型 API Key 会被视为敏感的本地凭据数据。

XplorOne 会通过 Electron `safeStorage` 或等效的系统级本地凭据保护机制保存模型凭据。

模型 API Key 不应以明文形式写入账本数据库。

非敏感的配置元信息可以保存在本地，例如：

- provider 名称
- base URL
- model name
- 启用状态
- 配置状态

这些元信息用于帮助 XplorOne 记住当前使用哪组模型配置，但不会把 API Key 当成普通账本数据处理。

更详细说明请见：[隐私与 AI 边界](./privacy-and-ai-boundaries_zh-CN.md)。

## 6. API Key 和本地 token 不是同一种东西

模型 API Key 和本地 API token、MCP token 不是同一种东西。

- **模型 API Key**
  用于把 XplorOne 连接到用户选择的大模型服务。

- **本地 API token**
  用于保护 XplorOne 本地 HTTP API 的访问。

- **MCP client token**
  用于授权 Codex、WorkBuddy 这类外部 agent 访问 XplorOne 的本地只读 MCP / query 路径。

BYOK 配置只处理模型 API Key。

本地 API 与 MCP token 会单独管理，而且普通桌面端使用不需要配置 MCP。

## 7. 替换或移除 API Key

你可以在模型服务设置里替换或移除模型 API Key。

以下情况可能需要这样做：

- API Key 已过期
- API Key 可能泄露
- 更换了模型服务商
- 想使用另一个模型
- 服务商账户或计费状态发生变化

如果 API Key 可能已经暴露，也请在对应模型服务商后台撤销或轮换该 Key。

## 8. 备份与迁移说明

XplorOne 的某些 full-app 备份工作流可能包含受保护的凭据元数据。

但是：

- 导出的账本文件不应暴露明文 API Key
- 受保护的凭据备份不等于可跨机器移植的明文 Key 导出
- 凭据恢复可能依赖当前机器或操作系统凭据环境
- 如果迁移到另一台机器，你可能需要重新配置模型 API Key

如果你在不同机器之间迁移 XplorOne 数据，建议迁移完成后重新填写或重新测试模型配置。

## 9. 常见配置问题

### 测试连接失败

请检查：

- API Key 是否正确
- base URL 是否正确
- model name 是否是该服务商支持的模型
- 服务商账户是否有足够额度或配额
- 服务商是否要求特殊 endpoint 格式
- 当前网络连接是否可用
- 当前系统代理或网络环境是否允许请求发出

### 保存后 AI 功能仍然不可用

保存配置和启用配置是两个动作。

请确认你已经启用当前配置。

### 服务商返回内容不符合预期

不同模型和服务商的行为可能不同。

有些服务商可能会：

- 返回普通文本，而不是严格 JSON
- 返回推理内容或包裹文本
- 需要特定请求参数
- 对 temperature 或结构化输出有特殊要求

XplorOne 会对部分模型行为做兼容处理，但不同服务商的表现仍可能存在差异。

### 回复语言不是你预期的语言

XplorOne 可能会综合判断：

- 你当前消息的语言
- 同一会话中上一条明确的用户语言
- 当前界面语言作为兜底
- 如果启用了 AI 个性化，也会使用与当前回复语言匹配的个性化上下文

如果语言表现不符合预期，反馈时建议提供：输入语言、界面语言、模型服务商和你期望的回复语言。

## 10. 安全建议

请不要：

- 分享你的 API Key
- 把 API Key 粘贴到公开 issue
- 把 API Key 提交到 GitHub
- 在截图里暴露 API Key
- 把 API Key 发送给第三方

如果你需要反馈问题，请先移除日志、截图和示例里的敏感信息。

如果 API Key 可能已经泄露，请在模型服务商后台撤销或轮换该 Key。

## 11. BYOK 不代表什么

BYOK 不代表：

- XplorOne 必须依赖 AI 才能使用
- XplorOne 内置免费模型额度
- XplorOne 控制你的模型服务商账户
- AI 可以自动写入账本
- 默认会把所有本地数据发送给模型服务商

BYOK 只是表示：

> 你可以选择并配置自己的模型服务，用于可选的 AI 辅助工作流。
