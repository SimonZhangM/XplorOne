# XplorOne 功能导览

XplorOne 是一款面向超级个体、自由职业者、一人公司经营者、顾问、独立开发者和小型工作室主理人的本地优先 AI 财务工作台。

这份文档用于帮助你快速理解 XplorOne 当前的主要功能区域。

它不是一步一步的安装教程。
如果你想完成第一次基础使用流程，请看：[快速开始](./getting-started_zh-CN.md)。

## 产品导航结构

当前 XplorOne 桌面端的主导航围绕以下区域展开：

- **记一笔**
- **首页**
- **日历**
- **列表**
  - 流水
  - 收支
  - 账户
- **报表**
  - 现金流
  - 收支
  - 资产负债
- **沟通 / Chat**
- **管理**
  - 预算
  - 成员
  - 项目

这个结构对应的是 XplorOne 的核心产品思路：

> 先记录财务活动，再通过列表、报表和 AI 辅助理解它。

## 1. 记一笔

记一笔是最快的流水录入入口。

它适合日常记账场景：你想快速记录一笔收入或支出，而不想打断当前工作流。

典型用途包括：

- 记录一笔支出
- 记录一笔收入
- 选择账户和分类
- 设置日期和备注
- 在需要时连续录入多笔流水

对于新用户来说，**记一笔通常是最适合开始的入口**。

使用记一笔不需要 API Key。

## 2. 首页工作台

首页是 XplorOne 的财务工作台。

它的作用是帮助你快速理解当前财务状态，并进入最常用的操作。

首页工作台通常可以呈现：

- 本月收入与支出概览
- 资产概览
- 每日收支趋势
- 支出分类排行
- 最近流水
- 预算提醒
- AI 助手快捷入口
- 记一笔、看流水、看报表、问 AI 等快捷操作

首页不是一个装饰性的仪表盘。
它是日常财务复盘和操作的起点。

## 3. 日历

日历提供按日期查看财务活动的方式。

它适合帮助你理解一个月中收入和支出发生的时间节奏。

典型用途包括：

- 按日期查看收入与支出
- 查看每日交易活动
- 定位某一天附近发生了什么财务记录
- 对比一个月中哪些天更活跃、哪些天更安静

当你更关心“什么时候发生”而不是“属于哪个分类”时，日历会很有帮助。

## 4. 列表

列表区域用于查看结构化财务记录。

它包括：

- 流水
- 收支
- 账户

### 流水

流水是交易记录列表，也是观察现金流动的主要区域。

你可以用它来：

- 查看流水记录
- 检查流入和流出
- 按时间范围筛选
- 查看账户相关活动
- 确认某笔记录是否保存正确

流水更关注“资金实际发生了什么流动”。

### 收支

收支页面帮助你从分类角度理解财务活动。

你可以用它来：

- 查看收入结构
- 查看支出结构
- 对比不同分类
- 查看分类层面的合计
- 跳转到相关流水记录

当你想知道“钱从哪里来”或“钱花到哪里去”时，这个页面会很有用。

### 账户

账户页面帮助你管理资产、负债和具体账户余额。

你可以用它来：

- 整理资产账户和负债账户
- 查看账户余额
- 理解账户结构
- 查看账户相关流水
- 区分不同资金空间

对于同时管理多个钱包、银行卡、信用卡、平台账户或经营账户的人来说，账户页面会很重要。

## 5. 报表

报表把原始财务记录转换成更高层次的财务视图。

当前报表导航包括：

- 现金流
- 收支
- 资产负债

在当前版本支持的范围内，报表与图表界面会对标题、范围、期间、金额、百分比、坐标轴单位、空状态和兜底名称做中英文显示处理。

### 现金流

现金流关注资金流动。

你可以用它来理解：

- 现金流入
- 现金流出
- 净现金流
- 月度变化
- 资金压力和节奏

如果你关心的是“钱有没有真的流进来、流出去”，现金流报表会很有价值。

### 收支报表

收支报表关注收入和支出的结构。

你可以用它来理解：

- 总收入
- 总支出
- 净收入
- 收支趋势
- 分类分布

这和流水中的现金流口径不同。
流水更关注资金流动；收支报表更关注收入与支出的结构。

### 资产负债

资产负债帮助你理解自己的整体财务状态。

你可以查看：

- 资产
- 负债
- 净资产
- 账户余额趋势
- 资产与债务结构

如果你不只是想看本月花了多少钱，而是想理解更长期的财务状态，这个页面会更有帮助。

## 6. 对话 / Chat

对话区域现在围绕两个助手展开：

- **本地助手**
- **AI 助手**

### 本地助手

本地助手用于快速本地工作流。

它可以帮助：

- 本地财务查询
- 流水查找
- 基础汇总
- 录入草稿工作流

受支持的本地查询和录入流程不会悄悄 fallback 到 AI 猜 intent。

### AI 助手

AI 助手用于更深入的解释和更开放的财务相关对话。

它可以帮助：

- 财务分析
- 趋势解释
- 期间对比
- 更开放的财务相关交流

AI 助手需要用户自行配置模型 API Key。

分析和更开放的财务相关对话会使用 XplorOne 专属助手设定，并明确数据边界、财务风险边界和回答纪律。

涉及写入的操作仍然需要用户确认。

更多说明可见：

- [BYOK 配置说明](./byok-setup_zh-CN.md)
- [隐私与 AI 边界](./privacy-and-ai-boundaries_zh-CN.md)

## 7. 管理

管理区域用于处理更结构化的财务管理对象。

它包括：

- 预算
- 成员
- 项目

### 预算

预算帮助你管理预期支出，并与实际支出进行对比。

你可以用它来：

- 创建月度预算
- 查看预算使用情况
- 比较实际支出和预算上限
- 理解预算剩余情况

预算不是第一次使用 XplorOne 的必要步骤。
但当你从“记录”进入“计划”时，它会变得很有用。

### 成员

成员用于管理和人员、角色有关的财务信息。

你可以用它来：

- 管理成员记录
- 在合适场景下关联流水
- 查看成员相关财务活动

这对家庭、小型工作室或项目型工作流会比较有帮助。

### 项目

项目用于按项目组织财务活动。

你可以用它来：

- 管理项目记录
- 将流水关联到项目
- 查看项目相关活动
- 区分不同业务线或工作线

对于自由职业者、顾问、小型工作室和一人公司来说，项目维度通常很实用。

## 8. 数据控制与可迁移性

XplorOne 包含数据管理能力，用于支持长期使用和本地掌控。

根据当前发布状态，数据相关能力可能包括：

- 备份
- 恢复
- 导出
- 导入
- 归档账本迁移
- `.xpl` 归档包工作流

这些能力服务于 XplorOne 的本地优先理念：

> 你的财务工作流应该可以迁移、可以恢复，并且始终在你的掌控之下。

数据与边界说明请见：[隐私与 AI 边界](./privacy-and-ai-boundaries_zh-CN.md)。

## 9. 本地 API 与 MCP

XplorOne 也包含面向 AI-native 工作流的本地集成能力。

这个区域是可选能力，不是普通桌面端使用的前提。

当前 MCP / query 路径主要面向 Codex、WorkBuddy 这类外部 agent。

关键边界包括：

- 本地 API 与 MCP 访问通过 token 保护
- MCP 当前为只读访问
- MCP 查询会经过 XplorOne 受控的本地 API 与查询链路
- MCP 不直接读取数据库
- 当前 MCP 路径不暴露写入能力

这使得 MCP 可以服务外部 agent 工作流，同时保持本地账本边界清晰。

更多说明请见：[隐私与 AI 边界](./privacy-and-ai-boundaries_zh-CN.md)。

## 10. 设置与个性化

设置用于调整 XplorOne 在你本机上的工作方式。

设置可能包括：

- 模型服务配置
- 模型服务商预设
- 显示偏好
- 主题模式
- 界面语言
- 个人资料
- 数据设置
- 提醒设置
- 已归档账本
- 本地 API 或 MCP 相关设置
- 授权或发布渠道相关设置

设置属于产品的控制层。
它决定 XplorOne 在你的机器上如何运行、如何显示、如何连接外部能力。

## 11. 当前产品边界

当前阶段，XplorOne 更适合被理解为：

- Windows 桌面发布线
- 本地优先财务工作台
- 面向超级个体、自由职业者、一人公司和小型工作室的产品
- 一个核心本地功能成立、AI 作为可选增强的财务工作台
- 一个有清晰数据边界和写入边界的桌面产品

XplorOne 当前不是：

- 云端优先的会计套件
- 面向所有地区、所有团队的大型企业财务系统
- 完整多平台发布
- 必须先配置 AI 才能使用的产品
- 允许 AI 不经确认就自动改写账本的系统

更多说明请见：[当前限制说明](./known-limitations_zh-CN.md)。
