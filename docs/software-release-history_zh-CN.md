# XplorOne v0.3.8

发布日期：2026-04-24

本次更新重点收口英文桌面体验、已有账本数据保护和设置页体验。XplorOne 现在会在不改写已有账本真实数据的前提下，更一致地本地化展示账户、分类、项目、模板、错误提示、菜单和列表文案，同时新增全局金额符号设置，完善模型配置，并开始支持常见英文记账句子的基础解析。

## Added 新增

- 新增全局金额符号设置，支持 `￥`、`$`、`€`。
- 新增模型预设持久化能力，保存模型配置时会记录所选预设 code。
- 新增模型设置状态展示，更清晰地区分已保存的草稿配置和当前真正启用的模型配置。
- 新增基础英文记账解析，支持常见收入、支出、转账、借款、还款、退款、账户、相对日期和 AM/PM 时间表达。
- 新增前端统一 API 错误本地化处理，覆盖常见数据、授权、归档、导入、恢复和设置失败提示。

## Changed 变更

- 英文界面将 `Ledger` 相关入口统一调整为 `Transactions`，覆盖导航、页面标题、操作按钮、确认提示和导出文案。
- 调整设置页侧边栏顺序，将日常偏好放在账本、模型、授权、数据和归档入口之前。
- 显示设置与提醒设置改为单一大卡片加内部分割线结构。
- 提醒设置的时间选择从原生时间输入框改为小时/分钟选择器，避免系统本地化造成中英文混排。

## Improved 优化

- 优化已有账本数据的显示层语言映射，包括账户名称、账户类型、收支分类、项目、成员/项目管理项、模板和 icon/emoji 展示。
- 优化英文月份标签，覆盖交易流水、账户、收支分类、预算、报表和日历等页面。
- 优化预算页展示，使月份胶囊、侧边栏月份、分类行和高亮分类都跟随当前语言和分类层级。
- 优化账户、收支分类、首页、报表、设置和个性化页面的间距、表格对齐、操作列位置和 tooltip 文案。
- 优化桌面菜单、关于弹窗和检查更新弹窗，补齐中英文文案，并展示当前版本层级和试用信息。
- 优化 Electron 桌面版模型请求链路，在可用时使用桌面网络能力发起请求。

## Fixed 修复

- 修复切换界面语言后账户名、分类名、项目名可能中英文混用的问题。
- 修复英文界面下部分收支分类和账户 emoji 映射缺失的问题。
- 修复语言切换后收支分类页和预算页可能出现重复或错误一级分类的问题。
- 修复预算页可能把二级分类展示到只应显示一级预算分类的位置。
- 修复设置页滚动方向异常，确保左侧侧边栏固定、右侧子页面滚动。
- 修复 seed 逻辑可能向已有数据账本重新补入默认账户、分类、成员或项目的问题。

## Security & Stability 安全与稳定性

- 降低升级期间误改已有账本数据的风险，默认数据 seed 只在空数据或明确旧默认数据场景下执行。
- 优化已有账本的语言切换方式，通过展示层本地化而不是向账本写入翻译后的重复数据。
- 强化模型配置存储，保存接口配置时同步保留预设元数据，提升模型设置恢复和启用流程的可靠性。

# XplorOne v0.3.7

发布日期：2026-04-24

本次更新重点优化了 AI 对话系统的中英文体验。现在 XplorOne 可以根据用户本次输入语言生成中文或英文回答，并提升了英文查询、本地解析和连续追问体验。

## Added 新增

- 新增对话系统语言上下文判断能力，可根据本次输入、同会话上一条明确语言和界面语言兜底决定回答语言。
- 新增基础英文查询本地解析能力，支持 `this month total income`、`how much did I spend this month`、`account balance`、`top expense categories this year` 等常见问题。
- 新增同一查询会话内的时间追问能力，例如 `and last month?`、`那上个月呢？` 会复用上一条查询并切换时间范围。

## Changed 变更

- AI 对话回答语言不再被界面语言强制决定，优先跟随用户本次输入语言。
- 查询、分析、自由对话与录入草稿的新生成文案开始按响应语言输出。
- 前端会随聊天请求传入当前界面语言，并根据后端返回的 `response_language` 渲染按钮语言。

## Improved 优化

- 优化查询回答、补问、表格标题、表格列名、金额与周期标签的双语展示。
- 优化分析与自由对话的系统提示词，使英文问题更自然地得到英文回答。
- 优化录入草稿的标题、字段名、缺失字段提示、确认、取消与提交结果的双语体验。

## Fixed 修复

- 修复英文界面下输入中文问题时，回答可能仍按英文输出的问题。
- 修复只追问时间范围时，系统无法复用上一条查询上下文的问题。

## Known Issues 已知问题

- 英文自然语言记账解析尚未纳入本轮更新，例如 `spent 20 on lunch from cash` 会在后续版本单独实现。

本次更新继续推进已归档账本的 `.xpl` 导出与导入能力。

## Added 新增

- 新增已归档账本 `.xpl` 单文件导出。
- 新增 `.xpl` 导入为新的活动账本。

## Improved 优化

- 优化 `.xpl` 包预检、文件名安全处理和重复导入体验。
- 优化导入时的 ID 重映射与一致性校验。

## Security & Stability 安全与稳定性

- `.xpl` 固定为 gzip 压缩后的 UTF-8 JSON 单文件，并限制包大小，降低异常文件风险。

# XplorOne v0.2.1

发布日期：2026-04-16

本次更新完成预算并库，把预算重新收口为主账本数据库中的业务域。

## Changed 变更

- `xplorone-budget.db / budget.db` 并入主库 `xplorone.db`。
- 预算页、首页预算摘要、快照、恢复、CSV 和未来 `.xpl` 都改为依赖主库预算表。

## Improved 优化

- 优化 legacy budget merge，旧预算库只作为一次性迁移来源。
- 优化预算并库元信息记录和幂等导入。

## Security & Stability 安全与稳定性

- 预算迁移采用主库事务，失败时不写完成标记并允许下次重试。

# XplorOne v0.2.0

发布日期：2026-04-16

本次更新正式进入 XplorOne 0.2 系列，重点推进 MCP、外部连接、归档迁移和产品化基础。

## Added 新增

- 新增 XplorOne Read-Only MCP v1 基线能力。
- 新增 Codex / WorkBuddy 客户端 token 管理基础。
- 新增仓库内 XplorOne skill 版本化内容。

## Changed 变更

- 外部 agent 查询优先走本地 API / QueryExecutor，不直接访问 SQLite。
- MCP 只开放只读查询，不开放写入和跨账本显式查询。

## Improved 优化

- 优化本地 API、MCP、设置页和外部工具说明之间的边界。
- 优化外部客户端结构化结果与文本通道兼容。

## Security & Stability 安全与稳定性

- MCP client token 通过 Electron `safeStorage` 加密保存。
- Local API 与 MCP 的生命周期开始区分“用户手动开启”和“MCP 临时租用”。

# 早期开发版本 v0.0.5 - v0.1.9

发布日期：2026-04-01 至 2026-04-16

这些版本主要是 XplorOne 从早期原型逐步形成桌面财务工作台的开发历史。为便于公开阅读，早期大量内部迭代在此合并为阶段性摘要。

## Added 新增

- 新增账户、分类、成员、项目、流水、首页概览和基础报表等核心财务模块。
- 新增 AI 查询、AI 录入、AI 分析和自由对话的早期链路。
- 新增账本设置、数据设置、预算管理和多账本基础能力。
- 新增专业图表能力，包括树状图、旭日图、瀑布图、双轴图、热点图等。

## Changed 变更

- 应用从单页原型逐步演进为多模块桌面工作台。
- 多个页面的导航、工具栏、图表和表格结构持续重排。
- 数据库、仓储、服务层和共享契约多次扩展，以支撑账本、流水、报表和 AI 对话。

## Improved 优化

- 持续优化快速记账、流水筛选、账户管理、分类管理、成员项目管理和报表展示。
- 持续统一页面视觉、图标、emoji、按钮、弹窗、筛选器和统计卡片。
- 持续增强模型适配、查询解析、时间范围解析和 AI 回答展示。

## Fixed 修复

- 修复多轮早期页面布局、图表显示、账户分类、流水筛选和统计口径问题。
- 修复部分历史 tag 重复指向同一提交的问题记录方式，避免虚构版本差异。

## Known Issues 已知问题

- 早期版本说明来自历史 tag 与 commit 摘要补录，细节不如后续版本完整。
