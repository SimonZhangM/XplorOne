# 数据保存与备份

这份文档说明如何理解 XplorOne 的本地数据、备份、导出、归档和换电脑迁移。

它不是恢复保证。用户仍应妥善保存自己的安全备份。

## 数据保存在哪里

XplorOne 被设计为本地优先的桌面财务工作台。

运行时数据保存在用户自己的本地应用数据环境中，可能包括：

- 账本
- 账户
- 分类
- 流水
- 预算
- 报表和派生视图
- 设置与本地元数据
- 本地集成配置

具体路径可能因操作系统、发布渠道和用户配置而变化。

## 备份可能包含什么

根据工作流和版本，备份可能包含：

- 财务记录
- 账本元数据
- 分类和账户
- 预算数据
- 归档元数据
- 设置
- 在支持范围内用于当前机器恢复的受保护凭据元数据

即使凭据没有以明文暴露，备份仍然可能包含敏感财务数据。

## 导出、归档和完整备份

不同数据工作流承担不同用途。

| 工作流 | 典型用途 | 说明 |
|---|---|---|
| 导出 | 移动或查看选定财务数据 | 不一定包含所有应用设置 |
| 归档 / `.xpl` | 保存或移动账本类数据包 | 范围取决于当前版本支持 |
| 完整备份 | 保存更完整的应用状态 | 可能包含受保护元数据和本地环境假设 |

不要默认这些工作流完全等价。

## API Key 与 Token

模型 API Key、本地 API Token 和 MCP Client Token 不应以明文形式出现在导出的账本文件中。

某些完整备份工作流可能包含用于当前机器恢复的受保护凭据元数据。受保护凭据备份不等于可跨机器移植的明文 Key 导出。

换电脑时，建议准备：

- 重新填写或重新测试模型 API Key
- 如有需要，重新生成本地 API 或 MCP Token
- 重新检查模型服务配置
- 确认导入或恢复后的财务数据正确

## 换电脑迁移

比较稳妥的迁移流程通常是：

1. 在旧电脑上创建当前备份或导出。
2. 保留一份未改动的备份副本。
3. 从官方来源在新电脑安装 XplorOne。
4. 恢复或导入数据。
5. 检查账户、分类、流水、报表和预算。
6. 如有需要，重新配置模型 API Key 和本地集成 token。
7. 在依赖新环境前，测试 AI、备份和导出工作流。

## 恢复和覆盖风险

恢复工作流可能改变本地状态。

恢复前请确认：

- 哪个账本或应用状态会受到影响
- 已为当前状态保留独立备份
- 不要用唯一一份真实财务数据测试恢复
- 恢复后先检查数据，再继续日常使用

## 云同步边界

XplorOne 当前不定位为自动云同步产品。

跨设备使用应通过备份、导出、归档，或未来明确发布的同步能力来管理。
