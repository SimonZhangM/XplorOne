# 常见问题

这份 FAQ 用于回答关于 XplorOne 发布范围、本地优先、AI 功能、API Key、MCP、数据边界和反馈支持的常见问题。

## 1. XplorOne 是什么？

XplorOne 是一款面向超级个体、自由职业者、一人公司经营者、顾问、独立开发者和小型工作室主理人的本地优先 AI 财务工作台。

它可以帮助用户记录财务活动、查看账户与报表、理解现金流，并在需要时使用 AI 辅助查询、录入草稿和分析。

## 2. XplorOne 适合谁？

XplorOne 适合需要自己管理财务工作流的人，例如：

- 自由职业者
- 顾问
- 独立开发者
- 小型工作室主理人
- 一人公司经营者
- 在意本地优先和财务可见性的用户

它不是面向所有团队、所有国家地区、所有合规场景的完整企业级会计系统。

## 3. XplorOne 是普通记账 App 吗？

不完全是。

XplorOne 包含记账能力，但它更接近一个财务工作台。

它的目标不只是记录流水，还包括帮助用户理解：

- 现金流
- 收入与支出
- 资产与负债
- 账户结构
- 分类结构
- 一段时间内的财务变化

## 4. 当前正式发布线是什么？

当前正式发布线是 Windows 桌面应用。

桌面端是面向终端用户的正式发布形态。

Web 预览只用于开发和内部预览，不是正式终端用户发布形态。

## 5. 使用 XplorOne 必须配置 API Key 吗？

不需要。

即使没有模型 API Key，XplorOne 也可以使用。

没有 API Key 时，你仍然可以使用这些核心本地功能：

- 本地记账
- 记一笔
- 账本、账户、分类和流水
- 报表与财务视图
- 基础本地查询
- 当前版本支持的备份、恢复、导出、导入和归档工作流

模型 API Key 只在你需要可选 AI 辅助能力时才需要。

## 6. AI 辅助能力包括什么？

AI 辅助能力可能包括：

- AI 助手中的分析与解释
- 更开放的财务相关对话
- 明确进入 AI 工作流时的模型辅助理解
- 启用模型参与时的录入草稿辅助

AI 是增强层，不是 XplorOne 的使用前提。

## 7. AI 会自动写入我的账本吗？

不会。

AI 不会自动写入账本。

涉及写入的操作必须由用户确认。

这尤其适用于：

- 创建流水
- 提交录入草稿
- 任何会改变账本数据的工作流

XplorOne 的设计目标是：本地助手可以处理受支持的本地查询和录入工作流，AI 助手可以辅助分析和更开放的财务相关对话。

是否写入仍然由用户确认。

## 8. 我的财务数据会默认上传到云端吗？

不会。

XplorOne 被设计为本地优先的桌面财务工作台。

核心财务记录保存在用户自己的本地应用数据环境中。

但当你使用 AI 辅助功能时，XplorOne 可能会把完成该任务所需的相关上下文发送给你自行配置的模型服务。

## 9. 哪些数据可能发送给模型服务商？

使用 AI 辅助功能时，XplorOne 可能会发送完成当前任务所需的信息。

根据具体工作流，可能包括：

- 用户当前的问题或指令
- 当前请求所需的结构化财务上下文
- 选定的报表或查询上下文
- 录入草稿上下文
- 最近对话上下文
- 与当前回复语言匹配的 AI 个性化摘要，仅在用户启用时使用

XplorOne 的目标是发送与任务相关的上下文，而不是默认把整个本地数据库发送给模型服务。

启用 AI 个性化时，XplorOne 的设计目标是发送受控的自然语言摘要，而不是原始设置 JSON。

用户应选择自己信任的模型服务商，并了解该服务商自己的数据与隐私条款。

## 10. 模型 API Key 是怎么保存的？

模型 API Key 会被视为敏感的本地凭据数据。

XplorOne 会通过 Electron `safeStorage` 或等效的系统级本地凭据保护机制保存模型凭据。

模型 API Key 不应以明文形式写入账本数据库。

详细说明请见：[隐私与 AI 边界](./privacy-and-ai-boundaries_zh-CN.md)。

## 11. 什么是 BYOK？

BYOK 是 **Bring Your Own Key** 的缩写。

在 XplorOne 中，它表示：你可以使用自己的模型 API Key，把 XplorOne 连接到你选择的模型服务。

BYOK 不代表 XplorOne 必须依赖 AI 才能使用。
它只表示：可选的 AI 辅助工作流可以使用你自己配置的模型服务。

应用可能提供 OpenAI、Google Gemini 等常见服务商预设，同时在支持范围内保留自定义 OpenAI-compatible 模型服务配置。

配置方式请见：[BYOK 配置说明](./byok-setup_zh-CN.md)。

## 12. 模型 API Key 和 MCP token 是同一个东西吗？

不是。

它们是不同类型的凭据：

- **模型 API Key** 用于把 XplorOne 连接到用户选择的模型服务。
- **本地 API token** 用于保护 XplorOne 本地 HTTP API 的访问。
- **MCP client token** 用于授权 Codex、WorkBuddy 这类外部 agent 访问 XplorOne 的本地只读 MCP / query 路径。

BYOK 配置只处理模型 API Key。

本地 API 与 MCP token 会单独管理，普通桌面端使用不需要配置 MCP。

## 13. 本地助手和 AI 助手有什么区别？

本地助手负责快速本地查询和录入工作流。

AI 助手负责分析和更开放的财务相关对话。

受支持的本地助手流程不会悄悄 fallback 到 AI 猜 intent；AI 助手会在需要 AI 能力时使用用户自行配置的模型服务。

## 14. XplorOne 里的 MCP 是什么？

MCP 是一个可选的本地集成路径，主要面向外部 agent 工作流。

它主要用于 Codex、WorkBuddy 这类工具。

当前 MCP / query 路径是：

- 本地的
- 通过 token 保护
- 只读的
- 围绕当前激活账本设计
- 通过 XplorOne 受控的本地 API 与查询链路执行

MCP 不是普通记账、查看报表、记一笔或软件内 AI 工作流的前提。

## 15. MCP 会直接读取数据库吗？

不会。

当前 MCP 访问会经过 XplorOne 受控的本地 API 与查询链路。

它不会直接读取数据库，也不会绕过应用的查询边界。

## 16. MCP 能写入或修改我的账本吗？

不能。

当前 MCP 路径是只读的。

它不暴露写入操作。

## 17. 备份和导出的文件可以公开分享吗？

不建议。

备份和导出文件可能包含敏感财务数据。

即使凭据没有以明文暴露，财务记录本身也是敏感信息。

请不要公开分享：

- 备份文件
- 导出的账本文件
- 带有财务数据的截图
- API Key
- 本地 token
- 模型凭据

如果你需要反馈问题，请先移除私人财务数据和凭据。

## 18. 导出的账本文件会包含我的 API Key 吗？

导出的账本文件不应暴露模型 API Key 或 MCP token 的明文。

某些 full-app 备份工作流可能包含受保护的凭据元数据，用于当前机器环境下的恢复，但受保护凭据备份不等于可跨机器移植的明文 Key 导出。

如果你把数据迁移到另一台机器，可能需要重新配置模型 API Key 或本地集成 token。

## 19. 我可以把 XplorOne 数据迁移到另一台电脑吗？

根据备份或导出工作流，财务数据可能可以迁移。

但受保护凭据可能依赖当前机器或操作系统凭据环境。

迁移到另一台机器后，你可能需要：

- 恢复或导入财务数据
- 重新配置模型 API Key
- 重新生成本地 API 或 MCP token
- 重新测试 AI 模型配置

## 20. XplorOne 支持所有银行或支付平台的导入格式吗？

不一定。

导入和导出覆盖范围会随版本变化。

用户不应默认所有银行、支付平台或会计软件导出格式都已支持。

导入后的数据应由用户自行检查。

## 21. XplorOne 是税务、法律或会计合规服务吗？

不是。

XplorOne 帮助用户整理、查看和理解财务记录。

它不是：

- 法律建议
- 税务建议
- 持证会计建议
- 特定国家或地区的税务申报系统
- 专业会计师或税务顾问的替代品

用户仍需自行判断财务决策，并遵守所在地的合规要求。

## 22. XplorOne 是开源软件吗？

不是，除非另有明确说明。

XplorOne 是专有软件。

这个仓库是公开产品入口，用于文档、版本发布、路线更新和社区反馈。它不是完整开源代码仓库首页。

详细说明请见 [LICENSE.md](../LICENSE.md)。

## 23. 我应该从哪里下载 XplorOne？

请使用本仓库或官方网站提供的官方下载链接。

GitHub 用户可以从 [GitHub Releases](https://github.com/SimonZhangM/XplorOne/releases) 开始。

请确认安装包来自官方来源。

## 24. GitHub Releases 和官网有什么区别？

GitHub Releases 可以作为公开版本发布和版本历史渠道。

官网可能提供产品页面、下载引导、文档链接和版本信息。

不同发布渠道的下载体验可能不同，但用户应始终使用官方链接。

## 25. 既然桌面端是正式发布线，为什么还有 Web 预览？

Web 预览用于开发和内部预览。

它不是正式终端用户发布形态。

一些桌面能力，例如系统文件选择器或本地桌面集成，在 Web 预览中可能不会以同样方式工作。

## 26. 为什么当前正式发布线是 Windows？

XplorOne 当前优先把 Windows 桌面版本打磨稳定、可用。

其他平台除非明确发布，否则不应默认为可用。

当前可用版本请查看 [Releases](https://github.com/SimonZhangM/XplorOne/releases)。

## 27. 如果 AI 回复语言不对，我应该怎么反馈？

反馈语言问题时，建议提供：

- 界面语言
- 输入语言
- 模型服务商
- 期望回复语言
- 实际回复语言
- 问题发生在本地助手还是 AI 助手

XplorOne 可能会综合判断当前消息语言、同一会话中的上一条明确用户语言，以及界面语言作为兜底。

如果启用了 AI 个性化，个性化上下文也应跟随当前回复语言。

## 28. 如果模型连接测试失败怎么办？

请检查：

- API Key 是否正确
- base URL 是否正确
- model name 是否是该服务商支持的模型
- 服务商账户是否有足够额度或配额
- 服务商是否要求特殊 endpoint 格式
- 当前网络连接是否可用
- 当前系统代理或网络环境是否阻止请求发出

更多说明请见：[BYOK 配置说明](./byok-setup_zh-CN.md)。

## 30. 提交 bug 时应该提供哪些信息？

一份有用的 bug 报告通常包括：

- XplorOne 版本
- Windows 版本
- 问题发生的位置
- 复现步骤
- 预期结果
- 实际结果
- 必要时的截图
- 是否涉及 AI、BYOK、本地 API 或 MCP

不要包含 API Key、token、私人财务数据或未脱敏的完整日志。
